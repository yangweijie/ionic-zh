卡片

卡片已经在最近几年广泛使用。它们是一个伟大的方式来包含和组织信息，同时建立一个用户可预计的预期。一次有这么多内容显示经常在一个不可变的小屏幕上，卡片已经很快变成许多公司选择的设计模式，包括像Google, Twitter, 还有 Spotify..

对于移动体验，卡片可以很容易直观地在许多不同尺寸的屏幕显示相同的信息。 它们允许更多控制，灵活，甚至可以是动画。卡片通常在另一个的上方，但是它们也可以用来像一个"page",然后左右滑动。

工具

Ionic自带了一可用的工具类名来帮助你快速美化你的设计。每一个都是可选的。

颜色

Ionic 自带一组颜色来上手，但是通常 颜色就是要被覆写的。我们更愿意说Ionic为你的颜色、色板、主题提供了一套推荐的命名转换。

工具颜色添加了用来帮助设定命名转换。你可以注意Ionic故意没有使用单词像 "red"或 "blue"，而是有代表一种情感或通用的主题颜色的方式命名。

让我们实际点，指定颜色是在CSS中比较容易的任务之一，而每个应用程序将有颜色不同的要求。Ionic的目标是提供一个干净的系统基础之上构建和维护，并保持远离要求了每个应用程序选择的颜色定制设计。

要定制颜色你可以简单覆写那些在ionic.css CSS文件中的样式。另外，既然 Ionic 是用Sass编译的,要更强大和灵活的你也可以 在 _variables.scss文件中修改和扩展颜色变量 。

内边距

Ionic的许多组件都有意将padding 和 margin 重置到了0. 许多情况下应用将会组件全屏，初始化时每个组件都是0边距开发者可以轻松控制应用中组件的padding 和 margins。

padding 工具类名可以被重用来给任何元素内容一些可喘息空间，同时默认在元素外框和内部内容间添加了一个15px 空隙。 下面的类名不被强制使用在任意元素上，但是也许对你的布局有帮助、

  • padding 每边添加 padding 。
  • padding-vertical 添加顶部和底部 padding 。
  • padding-horizontal 左右添加 padding 。
  • padding-top添加顶部 padding 。
  • padding-right 添加右部 padding 。
  • padding-bottom 添加底部 padding 。
  • padding-left 添加左侧 padding 。
